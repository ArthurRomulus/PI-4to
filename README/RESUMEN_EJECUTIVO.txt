            # ... crear inputs ...
            enable_keyboard_for_widget(self)  # ✅ Habilita para todos


3️⃣  CONTROL MANUAL (Si necesitas control explícito):
    
    from ui.keyboard_helper import KeyboardManager
    
    manager = KeyboardManager()
    manager.show_keyboard_for(input_field)  # Mostrar
    manager.hide_keyboard()                  # Ocultar

                           CARACTERÍSTICAS TÉCNICAS
═══════════════════════════════════════════════════════════════════════════════

✨ FUNCIONAMIENTO:
   • Event Filter: Detecta FocusIn/FocusOut en QLineEdit
   • Posicionamiento: Dinámico basado en tamaño de ventana
   • Timer: Delay de 150ms para cerrar sin flicker
   • Estado: Mantiene shift/caps y widget actual con focus

✅ SOPORTE PARA:
   • Números (0-9)
   • Letras mayúsculas/minúsculas (a-z, A-Z)
   • Espacios y caracteres especiales
   • Selección de texto y reemplazo
   • Múltiples inputs en mismo widget

═══════════════════════════════════════════════════════════════════════════════
                            RESOLUCIÓN DE PROBLEMAS
═══════════════════════════════════════════════════════════════════════════════

❓ El teclado no aparece
   ✅ Solución: 1. Verifica que es QLineEdit
                2. Asegúrate de enable_keyboard_for_widget()
                3. Reinicia la app

❓ El teclado no escribe
   ✅ Solución: 1. Verifica que el input tiene focus (borde azul)
                2. Revisa la consola para errores
                3. Reinicia app

❓ El teclado se cierra rápido
   ✅ Solución: Aumenta delay en ui/keyboard_manager.py línea 264:
                self._delay_close_timer.start(300)  # Cambiar 150 por 300
